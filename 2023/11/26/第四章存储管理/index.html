(只列举相关的知识点，定义啥的，以后再来补充)
引入：存储管理是计算机系统的重要组成部分，用于存储包括程序和数据在内的各种信息，亦属于非常重要的系统资源。

4.1程序的链接和接入

引入：要使源程序能够运行，必须经过：编译、链接和装入

4.1.1逻辑地址、物理地址和虚拟地址

  1. 逻辑地址(相对地址)
  2. 逻辑地址空间
  3. 物理地址
  4. 物理地址空间
  5. 虚拟地址空间和虚拟地址

4.1.2程序链接

链接程序在将几个模块装配成一个装入模块时，需解决以下问题：

  1. 修改模块的相对地址
  2. 变换外部调用符号

根据对目标模块的链接时间不同进行分类：

  1. 静态链接
  2. 装入时动态链接
  3. 运行时动态链接

4.1.3程序装入

装入后，要实行地址转换操作(被称为：地址映射、地址变换或重定位)
根据地址变换的时间进行分类：

  1. 静态重定位
  2. 动态重定位

4.2分区式存储管理

对内存采用连续分配方式

4.2.1单一连续分区存储管理

  1. 实现原理
    • 内存空间被分为系统区和用户区
    • 仅适合单用户单任务操作系统
  2. 分配与释放
  3. 地址映射
    • 静态重定位
  4. 单一连续分区的优缺点
    • 优点：
      1. 不需要硬件支持
      2. 管理简单，开销小，装入内存的程序仅由分配程序参与管理
      3. 安全性高，除了系统区以外，用户区仅有一个程序，不存在多个程序相互影响的问题
    • 缺点：
      1. 程序的地址空间受用户区地址空间的限制
      2. 由一个程序独占系统资源，会造成系统资源的严重浪费
      3. 不支持多用户

4.2.2固定分区存储管理

  1. 实现原理
    • 将内存用户空间划分为若干个固定大小的区域，每个区域称为一个分区，可以装入一个用户程序运行，分区一旦划分完成，便在整个运行期间保持不变。
  2. 分区划分
    • 分区大小相等
    • 分区大小不想等
  3. 数据结构
  4. 分配与释放
  5. 地址映射
    • 静态重定位
  6. 固定分区分配的优缺点
    • 优点：简单易行、CPU利用率较高
    • 缺点：程序大小受分区大小的限制、存储空间存在浪费

4.2.3可变分区存储管理

  1. 实现原理
    • 内存分区在程序运行时根据程序对内存空间的需要而动态建立，分区的划分时间、大小及位置都是动态的。
  2. 数据结构
    • 通常使用的数据结构包括：已使用分区表、空闲区表以及空闲分区链
  3. 分配算法
    • 最先适应算法
      • 循环最先适应算法
    • 最佳适应算法
    • 最差适应算法
  4. 分配与回收
    • 考虑对相邻的空闲分区进行合并
  5. 地址映射
    • 动态重定位
  6. 碎片问题
    • “紧凑”技术
