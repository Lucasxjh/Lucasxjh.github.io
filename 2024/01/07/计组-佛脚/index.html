第一章计算机系统概述

1.1计算机系统

  1. 存储程序控制的概念：
    • 1945 美籍 匈牙利 数学家 冯诺依曼提出
    • 两个子概念：存储程序，程序控制
      • 存储程序：程序和数据都存储在存储器中
      • 程序控制：用程序实现对计算的控制，完成指令的功能
    • 一条指令的执行过程：取指，分析，执行
    • 以“存储程序控制”为核心的计算机为“冯诺依曼”计算机
  2. 冯诺依曼计算机的特点
    • 硬件系统：运算器，存储器，控制器，输入输出设备
    • 以运算器为中心，输入输出设备与存储器之间的数据传送通过运算器完成
    • 计算机内部采用二进制，存储器存放的是指令和数据
    • 工作方式是控制驱动，控制驱动就是由指令流来驱动数据流，单指令流单数据流
  3. 计算机系统的组成
  4. 计算机的层次结构
    • 下层是上层的基础，上层是下层的拓展
    • 上下层可以分割，没有上层，只有下层也是可以的
    • 硬件和软件的分界面是操作系统
    • 软硬件的逻辑等效性：某一功能，既可以用软件实现，也可以由硬件实现
  5. 三种语言和三种程序
    • 机器语言：二进制代码语言，是计算机唯一可以直接识别和执行的语言
    • 汇编语言：用英文单词或其缩写（助记符）代替二进制的指令代码
    • 高级语言：更接近人类语言习惯的机器语言
    • 汇编程序：将汇编语言转换为机器语言
    • 解释程序：对高级语言一行一行翻译
    • 编译语言：将高级语言翻译转换为目标代码

1.2计算机主要的技术指标

  1. 与字长有关的技术指标
    • 字：CPU在同一时间内能作为一个整体被存取、传送、计算的一组二进制数
    • 机器字：计算机中二进制表示的数据或指令（CPU一次能直接处理的二进制数据）
      • 机器字长：CPU进行一次运算所能处理的二进制数据的位数。运算器的位数 = 通用寄存器的位数 = 数据总线宽度 = 机器字长
      • 机器字长：字长，字长越大，数据表示的范围越大，计算精度越高
      • 机器字长一般为字节的整数倍
    • 数据字：二进制表示的数据
    • 存储字：存储单元中二进制表示的数据或者指令
      • 存储字长：一个存储单元中二进制代码的位数
      • 存储字长 <= 机器字长
    • 指令字：计算机中二进制表示的指令
      • 指令字长：一个机器指令中包含的二进制代码的位数
      • 一条机器指令的总长度，指令字长取决于操作码、操作数地址的长度和操作数地址的个数
  2. 与执行时间、运算速度有关的指标（重点）
    • 主频：计算机内部主时钟的频率，单位：Hz、MHz，用“f”表示
    • 时钟周期：是CPU中最小的时间单位，单位：s、ms、ns，用“T”表示
      • 用时钟周期数表示计算机的时长
    • 注意：
      • 主频就是时钟频率
      • 时钟周期和时钟频率互为倒数
      • 主频为Hz，时钟周期为s
      • 主频为MHz，时钟周期为us，10 ^ -6
      • 主频为GHz，时钟周期为ns，10 ^ -9
    • 补充：
      • 指令周期包含若干个机器周期，机器周期包含若干个时钟周期
      • CPU周期就是机器周期
      • 中断周期是一种特殊的机器周期
    • 时间指标：
      • CPI时钟周期数：执行一条指令需要的时钟周期数（对一条指令而言）
        • 对于一段程序而言：CPI（平均） = 执行某段程序所需的CPU时钟周期总数 / 程序包含的指令条数
      • 指令周期：取出并执行一条指令所需的全部周期
        • 指令周期 = 时钟周期数 * 时钟周期
      • 平均指令周期：对一段程序或者一个机器而言，平均一条指令的执行时间
        • 平均指令周期 = 平均CPI * 时钟周期
      • CPU执行时间：执行一段程序所需要的总时间
        • CPU执行时间 = CPU时钟周期数 * 时钟周期（CPU时钟周期数：一段程序所包含的时钟周期数）
        • （指令条数 * 平均CPI） * 时钟周期
    • 运算速度指标：
      • 速度指标MIPS：每秒执行多少百万条指令
        • MIPS = 指令条数（百万） / 执行时间（秒）
        • MIPS = 指令条数 / （指令条数 * 平均CPI） / 主频
        • MIPS = 时钟频率 / 平均CPI * 10 ^ 6
        • MIPS = 时钟频率 / （平均指令周期 / 时钟周期 * 10 ^ 6）
        • MIPS = 1 / （平均指令周期 * 10 ^ 6）

第二章运算器

第一节各种信息的编码

  1. 非数值数据的编码
    • ASCII码：美国信息交换标准代码，7位二进制数表示，共128个字符
    • 字符串：连续的一串字符，通常方式下，字符串占用主存中连续的多字节，每个字节存放一个字符的ASCII码
    • 汉字码：包括汉字的输入编码，汉字内码，汉字字形码三种，它们是计算机中用于输入、内部处理和输出的三种用途的编码
    • 逻辑数据：一位二进制码的两个状态值分别表示
  2. 数值数据的编码
    • 真值：用“+”，“-”表示符号的数据
    • 机器数：符号数值化的数据表示方法，计算机内的数值数据的编码方法
      • 原码，反码：最高位为符号位
      • 补码：末位不变
      • -1和-128没有原码和反码
      • 移码：1为正，0为负（与一般情况相反），数值部分与补码相同
        • 移码全0时，对应真值的最小值-2^n；移码全1时，对应真值的最大值2^(n-1)-1
    • 定点数：约定数据的小数点位置是固定不变的
      • 注意：补码表示范围在负数比原码多一个数；补码表示的定点小数，无论多少位，最小负数都为-1；补码表示的定点整数（n+1位），最小负数都是-2^n
  3. 校验码
    • 定义：能够发现错误并纠正错误的数据编码
    • 原理：通过加一些冗余码，并且这些冗余码要和数据位发生某种关系，以便用来检验或纠错
    • 校验码包含k位有效数据位，和r位校验位
    • 码距：各合法码字的最小距离
    • 注意：
      • 检测e位错，码距d >= e + 1
      • 纠正t位错，码距d >= 2t + 1
      • 纠正t位错，同时检测e位错，码距d >= e + t + 1
    • 奇偶校验码
      • 奇校验：整个校验码中的“1”的个数为奇数
      • 能力：码距为2，检查奇数数的出错
      • 用途：常用于存储器读写检查，或数据传送过程中的检查
    • 海明校验码
      • 位数r：2^r >= k + r + 1
      • 位置：校验位Pi应放在海明码的2^(i-1)的位置上，其余各位为数据为Di，从低到高依次排放
      • 关系：被校验的每一位位号要等于校验它的各检验位的位号之和

第二节定点数的运算

  1. 定点数的加减运算
    • 定点数的补码运算：任意两个数的补码之和，等于该两个数和的补码
    • 定点数的补码减法：任意两个数的补码之差，等于被减数的补码加上减数的变补
  2. 溢出的判断
    • 单符号位判断
      • 同号相加，结果符号位与参与运算的操作数符号相反，则溢出
      • 异号相减，结果符号位与被减数的符号相反，则溢出
    • 双符号位判断（最高位位真正符号位）
      • 双符号位表示：正数“00”，负数“11”
      • 如果运算结果两符号位不同，则溢出
    • 进位判断
      • 两个重要进位：最高数值位向符号位的进位；符号位向更高位的进位
      • 若两个进位同时存在，或同时不存在，就不溢出，只有一个存在则溢出
  3. 定点数的移位运算
    • 逻辑移位：补0（原码乘除运算中都是逻辑移位）
    • 算术移位：保持符号位不变。（符号位不变，移出位丢弃，对于正数：左右都补0；对于负数：原码左右都补0，反码左右都补1，补码左补0右补1）
    • 左移可能溢出，右移可能会影响精度
  4. 定点数的乘除运算
    • 原码：符号位不参与运算，单独处理
    • 补码：符号位参与运算，结果位补码
    • 定点数乘除运算
      • 原码乘法
        • 逻辑右移
      • 补码不恢复余数法

第三节ALU和定点运算器

第四节浮点数的表示与运算

  1. 浮点数的表示
    • 规格化浮点数：原码规格化的尾数最高位一定为1；补码规格化数的尾数最高位一定与尾数符号位相反
    • 左规：运算结果的尾数的最高位不是有效位时，尾数左移1位，阶码减1
    • 右规：运算结果的尾数的有效位进到小数点前面时，尾数右移1位，阶码加1
    • IEEE754标准
      • 短浮点数：32位；数符1位，阶码8位，尾数23位，偏置值127
      • 长浮点数：64位；数符1位，阶码11位，尾数52位，偏置值1023
      • 临时浮点数：80位；数符1位，阶码15位，尾数64位，偏置值16383
      • 尾数原码，且为规格化数，阶码为移码
  2. 浮点数的运算
    • 浮点数的加减运算
      • 阶码溢出，才算作溢出
    • 浮点数的乘除运算

第三章存储系统（最重要）

第一节存储系统概述

  1. 存储系统的层次结构
    • 主要包括：缓存-主存和主存-辅存两个存储层次
    • CPU和缓存、主存直接交换信息
    • 赋存和CPU不能直接交换信息，只能和主存交换信息
    • 设置目的：解决存储器大容量，低价格，高速度三者之间的矛盾关系
  2. 存储器的分类
  3. 存储器的性能指标
    • 存储容量 = 存储字数 * 字长
    • 编址方式：按字节编址，按字编址，按半字编址，按双字编址（字为机器字）
    • 存储速度：数据传输率 = 数据的宽度（字节） / 存储周期
    • 存储器的存放方式
      • 大端方式：高字节低地址
      • 小端方式：高字节高地址
      • 按边界对齐

第二节主存储器

  1. 主存储器的组织
    • 主存储器：由存储体，地址译码驱动电路，I/O和读写电路组成
    • MAR：存储器地址寄存器，存放的是CPU访问内存的地址
    • MDR：存储器数据寄存器，存放的是CPU和内存交换的数据
    • 地址译码方式
      • 单译码方式（字选法）
        • 仅有一个地址译码器
        • 若输入线为K，输出线为2^K
        • 2^K 条字线对应M = 2^K个存储单元
      • 双译码方式（重合法）
        • 把K位地址线分成接近相等的两段，一段水平，一段垂直
  2. 静态随机存储器SRAM
    • 六管静态MOS存储单元组成的双稳态电路，利用触发器来保存信息
    • 存取速度快，集成度低，功耗大，价格昂贵，一般用于高速缓存存储器
  3. 动态随机存储器DRAM
    • 利用栅极电容上的电荷来存储信息
    • 易于集成，位价低，容量大，低功耗，速度慢，一般用于大容量的主存储器
    • 地址复用，分时传送是动态RAM的特点
    • 刷新操作
      • 三种刷新方式：集中刷新、分散刷新、异步刷新
        • 集中刷新：
          • 在规定的一个刷新周期内，对全部存储单元集中一段时间逐行进行刷新，此刻必须停止读/写操作
          • 刷新时间 = 存储矩阵行数 * 存储周期
          • 存在“死时间”，集中刷新的时间
          • 死时间率 = 死时间 / 刷新周期
        • 分散式刷新：
          • 把刷新操作分散到每个存储周期内进行
          • 存储周期 = 读或写周期 + 刷新一行的时间（刷新一行的时间 = 存储周期）
          • 通常情况下没有死时间
        • 异步式刷新：
          • 把存储阵列的每行分散到2ms内刷新，但不集中刷新，是平均分配
          • 相邻两行的刷新间隔 = 最大刷新间隔时间 / 行数
          • 死时间就是一个读写周期
  4. 主存储器与CPU的链接（重点）
    • 三种扩展方式
      • 位扩展：CPU的数据线数与存储芯片的数据位数不相等，增加存储字长
      • 字扩展：增加存储器中字的数量（单元个数），但位数不变
      • 字位同时扩展：所有芯片被分为若干组，组内位扩展，组间字扩展，CPU每次访问的是一组中的所有芯片
    • 连接方法
      • 地址线的连接
        • CPU地址线低位连芯片，高位做片选
      • 数据线的连接
        • CPU的数据线进和存储芯片的数据线相等直连，不等按片连
      • 读/写控制线的连接
        • CPU的读写控制线和存储芯片的读写控制线直接连接
      • 片选线的连接
        • 存储芯片是否被选中取决于该存储芯片是否接收到来自CPU的片选有效信号
        • 线选法：用高位地址线直接分别连接至各个存储芯片的片选端
        • 译码片选法：用高位地址通过地址译码器芯片产生片选信号
  5. 提高主存速度的奇数
    • 双端口存储器
      • 具有两个彼此独立的读/写口
      • 每个读/写口有一套独立的地址寄存器和译码电路，可并行地独立工作
      • 两个口可以按各自接收的地址，同时读出或写入，或一个写入而另一个读出
      • 若两个口同时操作存储器的同一存储单元，将会发生冲突
      • 可以同时读，但不能同时写操作
    • 相联存储器
      • 按所存记录的全部内容或部分内容进行查找，并将相关内容进行读出或写入
      • 应用场景：存放虚拟存储器的段表、页表和快表；或者用于存放Cache的行地址
    • 多体交叉存储器
      • 多个模块组成的存储器，每个模块有着相同的容量和存取速度，每个模块都可以看成一个额独立的存储器
      • 各个模块具有各自的地址寄存器MAR，数据寄存器MDR，读/写电路和译码电路
      • 两种地址编址方式
        • 高位交叉编址：
          • 地址寄存器的高位表示体号，低位表示体内地址（顺序方式）
          • 体内地址连续，一个存储体存满，再存入下一个存储体
          • 可以在同一时间使得不同的请求源同时访问不同体
        • 低位交叉编址
          • 低位表示体号，高位表示体内地址
          • 同一存储体是不连续的，是以n为模交叉编址的
          • 连续的程序或数据交叉地存放在n个存储体中，以n为模交叉并行存取
          • 访存冲突概率减少，提高主存带宽
          • 同一请求源并行地访问不同的存储体
